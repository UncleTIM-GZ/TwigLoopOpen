# Public Beta 准入与边界定义

**适用版本:** v1.0.0-beta
**日期:** 2026-03-15

---

## Beta 目标

在更大规模的真实用户群中验证 MVP 主链路的可用性和稳定性：

**发起项目 → 结构化任务 → 协作者申请 → 试协作启动 → 7 天内出现进度信号**

相比 Pilot（≤ 20 人），Public Beta 扩大到 100 人规模，验证并发承载、用户体验和运维流程。

---

## 开放策略

| 维度 | 策略 |
|------|------|
| 准入方式 | **邀请制 + 白名单** |
| 邀请渠道 | 项目团队直接邀请 / 现有用户推荐 |
| 白名单管理 | 后端配置，支持按邮箱或域名批量添加 |
| 自助注册 | **不开放**，需持有效邀请码 |

---

## 用户规模

| 维度 | 限制 | 说明 |
|------|------|------|
| 总用户数 | **≤ 100** | 白名单控制 |
| 同时在线 | **≤ 50** | 基于单实例 core-api 承载评估 |
| 活跃项目 | **≤ 20** | 含 general / public_benefit / recruitment 三种类型 |
| 单项目协作者 | **≤ 10** | 席位唯一约束保护 |

---

## 功能边界

### 已开放 (Beta 可用)

| 功能 | 说明 |
|------|------|
| 注册 / 登录 | JWT + bcrypt，CORS 保护 |
| 用户资料 CRUD | ProfileService 三层架构 |
| 项目创建与发布 | 三种类型，状态机保护 (draft → published → ...) |
| 工作包 / 任务卡 | EWU/RWU/SWU 自动计算，状态保护，白名单过滤 |
| 申请与席位管理 | 状态机 + 唯一约束 + 并发控制 (SELECT FOR UPDATE) |
| VC 凭证签发 / 验证 | Literal type 约束，Rate limiting |
| 事件记录 | NATS JetStream + 域事件日志 |
| 前端全流程 | 首页 → 项目列表 → 详情 → 申请 → 工作台 |
| MCP Server | 标准 MCP 协议，AI 辅助功能 |

### 暂不开放

| 功能 | 状态 | 原因 | 预计开放 |
|------|------|------|----------|
| GitHub Source Sync 真实接入 | 代码就绪，未接 webhook | 需配置 GitHub App + webhook 安全验证 | GA |
| Temporal Workflows | 代码就绪，未启用 | 增加运维复杂度，需稳定运行基础 | GA |
| ClickHouse 分析 | 代码就绪，无消费者 | 等 Beta 产生足够数据后评估 | GA |
| Admin / Operator 管理后台前端 | 只读 API 就绪 | 管理后台前端尚未实现 | GA |
| 公益项目前端完整流程 | API 就绪 | 复核人 / Sponsor 前端流程不完整 | GA |

---

## 数据策略

| 策略 | 说明 |
|------|------|
| 数据持久性 | Beta 数据**可能重置**，不承诺迁移到 GA 环境 |
| 数据库备份 | 每日 `pg_dump` 自动备份 |
| 个人数据 | 不处理真实个人敏感数据（身份证、银行卡等） |
| 数据清理 | Beta 结束后，用户可申请删除个人数据 |
| 重置通知 | 如需重置数据，提前 72 小时通知所有 Beta 用户 |

---

## 环境边界

| 环境 | 用途 | 状态 |
|------|------|------|
| `local` | 开发环境 (Docker Compose) | 活跃 |
| `dev` | Beta 运行环境 | **活跃** |
| `staging` | GA 前预发布验证 | 未启用 |
| `prod` | 正式生产环境 | 未启用 |

---

## Beta 退出条件

出现以下情况时可终止 Beta：

1. 发现影响数据完整性的 P0 bug 且无法在 24 小时内修复
2. 安全漏洞被利用，用户数据泄露
3. 基础设施连续不稳定（72 小时内 ≥ 3 次服务不可用）
4. 用户反馈主链路（创建→申请→协作）系统性不可用

| 9 | 分页补齐所有列表端点 | [ ] |
