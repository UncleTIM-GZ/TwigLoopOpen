# Public Beta 入门指南

**适用版本:** v1.0.0-beta
**日期:** 2026-03-15

---

## 平台简介

Twig Loop 是一个 AI 原生的项目协作平台。用户可以发起项目（通用、公益或招募类），将项目结构化为工作包和任务卡，匹配协作者，并通过 GitHub 等外部平台追踪真实进度信号。当前处于 Public Beta 阶段，核心链路已通过安全加固和功能验证。

---

## 注册与登录

### 注册

1. 获取 Beta 邀请码（通过官方渠道或已有用户邀请）
2. 访问平台首页，点击「注册」
3. 填写邮箱、密码（≥ 8 位）、显示名称
4. 注册成功后自动获得 `access_token`，即可使用平台

### 登录

1. 访问首页，点击「登录」
2. 输入注册邮箱和密码
3. 登录成功后进入工作台

> **注意:** 当前版本使用 JWT 认证。请勿在公共设备上保持登录状态。

---

## 创建项目流程

1. **进入工作台** — 登录后点击「创建项目」
2. **填写基本信息** — 项目标题、描述、项目类型（`general` / `public_benefit` / `recruitment`）
3. **创建工作包** — 每个项目至少一个工作包，工作包是任务的组织容器
4. **创建任务卡** — 在工作包下创建具体任务，设置任务类型、复杂度、关键性等参数
5. **EWU 自动计算** — 系统根据任务参数自动计算工作量单位
6. **发布项目** — 确认无误后发布，项目变为公开可见

```
项目生命周期: draft → published → in_progress → completed / closed
```

---

## 申请协作流程

1. **浏览项目列表** — 在首页或项目列表中查找感兴趣的项目
2. **查看项目详情** — 了解项目描述、工作包结构、任务要求
3. **提交申请** — 点击「申请协作」，填写申请信息
4. **等待审核** — 项目发起人审核申请
5. **获得席位** — 审核通过后获得项目协作席位，可开始工作
6. **领取任务** — 在工作台中领取具体任务卡

```
申请生命周期: pending → approved / rejected
```

## 工作量单位说明

### EWU (Effort Work Unit) — 基础工作量

所有任务都有 EWU，衡量任务的工作量大小。

```
EWU = 任务类型基础权重 × 平均(复杂度, 关键性, 协作复杂度) × 风险乘数
```

- 任务类型权重: 0.6 (协作支持) ~ 1.5 (开发)
- 风险乘数: 1.0 (低) / 1.3 (中) / 1.6 (高)
- 单任务上限: **8 EWU**

### RWU (Reward Work Unit) — 招募激励

仅招募类项目 (`has_reward=True`) 自动计算:

```
RWU = EWU × 1.2
```

### SWU (Sponsor Work Unit) — 公益激励

仅公益 sponsor 项目 (`has_sponsor=True`) 自动计算:

```
SWU = EWU × 1.0
```

> 详细规则参见 `docs/ewu-rwu-swu-rules.md`。EWU 变更时 RWU/SWU 自动联动重算。

## 当前开放功能

| 功能 | 说明 |
|------|------|
| 注册 / 登录 | JWT + bcrypt，含 CORS 保护 |
| 用户资料 CRUD | ProfileService 三层架构 |
| 项目创建与发布 | 三种项目类型，状态机保护 |
| 工作包 / 任务卡 | EWU 自动计算，状态保护，白名单过滤 |
| 申请与席位管理 | 状态机 + 唯一约束 + 并发控制 |
| VC 凭证签发 / 验证 | 类型约束 + Rate limiting |
| 事件记录 | NATS JetStream + 域事件日志 |
| 前端全流程 | 首页 → 项目列表 → 详情 → 申请 → 工作台 |
| MCP Server | AI 辅助功能，标准 MCP 协议 |
| 公益 / 招募项目 API | 后端 API 完整支持 |

## 当前不开放功能

| 功能 | 状态 | 说明 |
|------|------|------|
| GitHub Source Sync | 代码就绪，未接真实 webhook | 需配置 GitHub App，Beta 期间不启用 |

| Admin / Operator 管理后台 | 后端只读 API 就绪 | 管理后台前端未实现 |
| 公益项目前端完整流程 | API 就绪，前端基础 | 复核人/Sponsor 前端流程不完整 |

## 已知限制

| 类别 | 限制 |
|------|------|
| 用户规模 | 邀请制，≤ 100 用户 |
| 活跃项目 | ≤ 20 个 |
| 单项目协作者 | ≤ 10 人 |
| 数据持久性 | Beta 数据可能重置，不承诺迁移到 GA |
| JWT 存储 | 当前使用 localStorage，后续迁移 httpOnly cookie |
| 分页 | 部分列表端点分页补齐中 |
| 日志 | 无结构化日志和 request_id 追踪（GA 前完成） |
